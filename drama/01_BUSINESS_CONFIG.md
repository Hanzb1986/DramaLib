## 一、每日自动任务流程（10:00 / 20:00）

```
Step 1: web_search 搜当前最火的短剧话题
Step 2: AI 创作脚本（Markdown格式，含口播+字幕+画面描述）
Step 3: 调用 lh-video-gen 生成竖版短视频
Step 4: 通过蚁小二上传并发布到目标平台
Step 5: 记录到 MEMORY.md + 02_RUNTIME_INDEX_LOG.md
```

### 质量兜底
如果今天的热点不够好或者脚本太差 → **跳过不发，不凑数**。宁可少发一条，也不发烂片。

## 二、脚本规范

```
## 开场（Hook）
**口播**：3秒抓人，用悬念/反转让观众停下来
**字幕**：大字标题

## 剧情1（矛盾出现）
**口播**：讲核心冲突
**字幕**：关键金句

## 剧情2（高潮）
**口播**：反转/高潮点
**字幕**：吸睛文案

## 结尾（引导）
**口播**：总结+引导关注
**字幕**：关注引导
```

## 三、视频参数

- 分辨率：1080×1920（9:16竖屏）
- 时长：60~90秒
- 口播语速：+10%
- TTS音色：zh-CN-YunxiNeural

## 八、Nova 协同规则（摘要）

> 以下规则与全局统一，完整内容参考 Nova 的 HEARTBEAT.md 和相关 Skill。此处仅列关键执行点。

### 8.1 任务转达

每次会话结束后，自检是否产生新任务（Hans 交代的、自己发现的、分析中识别出的）。有则立即 `sessions_send(agentId:"main")` 转达 Nova，不得等待巡检。

### 8.2 查询响应

收到 Nova 的查询消息 → 最高优先级，≤30 秒内准确答复。

### 8.3 优先级

| 优先级 | 响应 |
|:------|:----|
| 🔴特急 | 打断当前工作，立即处理 |
| 🟡紧急 | 完成当前步骤后，下一个处理 |
| 🟢正常 | 按节奏排队 |

### 8.4 冲突裁决

Nova 告知产出与其他 Agent 矛盾时 → 不自辩，等 Hans 裁决。

### 8.5 告警

| 类型 | 标识 | 触发 |
|:----|:----|:----|
| 通信故障 | 🔴 | sessions_send 失败/文件读写异常/网络不可达 |
| 技能缺失 | 🟡 | 所需 skill 未找到/路径错误 |
| 任务逾期 | 🟠 | 超出截止时间 |
| 定时失效 | ⚪ | cron 停跑/heartbeat 未唤醒 |
| 知识库冲突 | 🔵 | 规则版本不一致/记忆矛盾 |

发现即报，格式：`sessions_send(agentId:"main", {type:"alert", category:"🔴", ...})`

### 8.6 咬尾防御

Nova→你→处理完→通报「已完成」→Nova 回复「收到」→**你停止回复**。砍掉无意义回声，保留执行中的正常交流。

### 8.7 门禁（任务完成三步检查）

```
Step 1  Nova 通知发了吗？→ 否则立即发
Step 2  运行日志写了吗？→ 否则更新 02_RUNTIME_INDEX
Step 3  当日记忆更新了吗？→ 否则写 memory/YYYY-MM-DD
三门全过 → 汇报给 Hans
```

### 8.8 跨 Agent 边界

禁止直接 write/edit Ellie/Linna 的工作区文件。如需修改 → 上报 Nova 派工。

### 8.9 能力变更上报

Skill 新增/修改/删除 → 立即上报 Nova。
