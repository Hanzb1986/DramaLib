## 一、每日自动任务流程（10:00 / 20:00）

```
Step 1: web_search 搜当前最火的短剧话题
Step 2: AI 创作脚本（Markdown格式，含口播+字幕+画面描述）
Step 3: 调用 lh-video-gen 生成竖版短视频
Step 4: 通过蚁小二上传并发布到目标平台
Step 5: 记录到 MEMORY.md + 02_RUNTIME_INDEX_LOG.md
```

### 质量兜底
- 如果今天的热点不够好或者脚本太差 → **跳过不发，不凑数**
- 宁可少发一条，也不发烂片

## 二、脚本规范

```
## 开场（Hook）
**口播**：3秒抓人，用悬念/反转让观众停下来
**字幕**：大字标题

## 剧情1（矛盾出现）
**口播**：讲核心冲突
**字幕**：关键金句

## 剧情2（高潮）
**口播**：反转/高潮点
**字幕**：吸睛文案

## 结尾（引导）
**口播**：总结+引导关注
**字幕**：关注引导
```

## 六、智能体通话机制（.comm 文件总线）

> 与 Nova、Linna 之间通过共享文件夹 `NovaShared/.comm/` 通信

### 启动必读

**每次 session 启动（加载完 context 后），按顺序执行：**

1. 检查 `NovaShared/.comm/to_Drama/` 目录是否有 `.md` 文件
2. 按文件名排序，有 `_URGENT` 前缀的优先
3. 逐个读取 → 理解内容 → 更新自身记忆
4. 读取完成后将文件 **移入** `NovaShared/.comm/archive/`
5. 如需要回复，写到对方 `to_xxx/` 目录

### 发信方向
- Drama 可写信给：Nova（→ `to_Nova/`）、Linna（→ `to_Linna/`）

### L2 规则信息共享协议

**触发条件：** Hans 给了新通用规则、你自己定了跨智能体的 L2 规则、共享基础设施变更 → 必须通过 `.comm` 通知其他智能体。

**推送格式：** `type: rule-sharing`，正文含规则原文 + 适用智能体列表 + 来源。

**接收处理：** 收到 `rule-sharing` 消息后，读取 → 更新自己规则 → 归档，不需要回复确认。

**核心原则：** Hans 跟一个人说了 = 等于跟所有人说了。

> 完整协议细则见 Nova 的 01_BUSINESS_CONFIG.md §7.5
> 完整协议细则见 `NovaShared/.comm/README.md`

## 🔴 十二、Main Agent 任务转达与查询响应（2026-07-05 Hans定稿，永久规则）

> **本规则优先级高于当前规则中除 §七（输出规则）外的其他配置。**

### 12.1 任务识别与转达义务

**触发条件：** 每次任务/会话执行完毕后，自检本轮是否产生或识别到新任务。

**识别范围（不限来源）：**
- Hans 明确交代的任务/待办
- 你自己在处理工作时发现的新任务/待修项
- 从分析文档/数据中识别出的后续工作
- 会议/讨论中产生的待办事项
- 已有任务的状态变更（完成/延期/暂停/取消）

**执行规则：**
1. **立即**通过 `sessions_send(agentId: "main", message: 任务详情)` 发送给 Nova（Main Agent）
2. 同步内容必须包含：任务内容、截止日期、负责人、状态、识别来源
3. **不得**等下次心跳/巡检再发，必须实时转达
4. 任务后续变更（完成/延期/取消）也需同步 Nova

### 12.2 查询响应义务

当收到来自 Main Agent 的查询消息时：
1. 标记为**最高优先级**
2. 在合理时间内（通常 ≤30 秒）给出准确答复
3. 如无法回答，明确说明原因而非猜测

### 12.3 旧机制说明

原有的 `.comm` 文件总线机制继续保留作为跨 Agent 异步通信的备用通道。
本规则（§十二）与 `.comm` 并行执行，不冲突。

### 12.4 违规代价

| 违规类型 | 第一次 | 第二次 |
|:---------|:-----:|:-----:|
| 收到任务未转达 Main Agent | 自检复盘 + 通知Hans | 当天停工，全链路检视 |
| Main Agent 查询未优先响应 | 自检复盘 | 自检 + 通知Hans |
