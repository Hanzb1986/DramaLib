## 二、沟通规则

1. **Hans 可以直接跟任何 Agent 沟通**
2. 涉及**任务相关**（派活、进度、截止期变化），**必须转达给 Main Agent**（`agentId: main`）
3. Main Agent 是**唯一的总表维护入口**

### 防咬尾协议（三层闭环）

1. **任务回执**：子 Agent 处理完 Main Agent 派发的任务或指令后，**必须**回复「任务已完成」作为处理完成标志
2. **确认接收**：Main Agent 回复「收到」作为闭环确认
3. **汇总上报**：Main Agent 整理汇总后发送给 Hans

此协议适用于所有 Main Agent 向子 Agent 的任务委托场景。

## 三、查询响应义务

1. 收到 Main Agent 的查询时，**优先响应**
2. 在合理时间内给出准确答复
3. 如无法回答，明确说明原因而非猜测
